# Perguntas Frequentes

---

??? question "Preciso saber programar para usar este material?"
    **Não.** Este é exatamente o ponto do dr.dev. O conteúdo foi pensado para
    profissionais que não têm experiência com programação. As ferramentas que
    usamos permitem criar aplicações usando linguagem natural.

    Se você sabe escrever instruções claras (como um memorial descritivo ou um
    escopo de projeto), você já tem a base necessária.

??? question "Quanto custa usar essas ferramentas?"
    A maioria das ferramentas tem **planos gratuitos** que são suficientes para
    aprender e criar projetos iniciais:

    - **Claude** (claude.ai): uso gratuito generoso
    - **ChatGPT**: plano gratuito disponível
    - **Bolt** (bolt.new): gerações limitadas por dia no plano gratuito
    - **GitHub Pages**: hospedagem gratuita para sites estáticos
    - **Vercel**: plano gratuito para projetos pessoais

    Conforme seus projetos crescerem, pode fazer sentido assinar planos pagos
    (geralmente entre US$10-20/mês), mas para começar não é necessário gastar nada.

    O **Claude Code** (ferramenta de terminal) requer um plano Claude Pro (US$20/mês)
    ou créditos de API.

??? question "Quanto tempo leva para aprender?"
    A trilha completa tem **7 módulos** com tempo estimado total de 4-5 horas.
    Você pode completar no seu ritmo -- não precisa fazer tudo de uma vez.

    Depois de completar os 3 primeiros módulos (cerca de 2 horas), você já terá
    uma base sólida para começar a experimentar por conta própria.

??? question "É seguro usar IA para criar aplicações?"
    Algumas considerações importantes:

    - **Não coloque dados sensíveis** em prompts (senhas, dados de clientes, informações confidenciais da empresa)
    - **Sempre revise o resultado** -- a IA pode cometer erros, assim como qualquer ferramenta
    - **Para projetos críticos**, consulte um desenvolvedor profissional
    - **Para protótipos e ferramentas internas**, o risco é baixo e o benefício é alto

    Pense como qualquer ferramenta de engenharia: use com critério, verifique os resultados,
    e envolva especialistas quando a complexidade ou o risco justificarem.

??? question "Posso usar isso profissionalmente?"
    **Sim**, com ressalvas. AI coding é excelente para:

    - Protótipos e provas de conceito
    - Ferramentas internas para sua equipe
    - Automatização de tarefas repetitivas
    - Dashboards e relatórios visuais

    Para sistemas de produção que vão atender clientes externos ou lidar com
    dados críticos, é recomendável envolver desenvolvedores profissionais --
    mas o protótipo que você criou com IA pode servir como especificação
    excelente para a equipe de desenvolvimento.

??? question "E se a IA gerar algo errado?"
    Isso vai acontecer. É normal e esperado. A IA não é perfeita.

    A boa notícia: você pode simplesmente dizer "isso não está certo" e explicar
    o que precisa mudar. O processo é iterativo. Cada correção melhora o resultado.

    Lembre-se da analogia com engenharia: você também revisa o trabalho de uma
    construtora e pede ajustes. O mesmo vale para a IA.

??? question "Qual a diferença entre Claude, ChatGPT e Gemini?"
    Todos são assistentes de IA capazes, mas têm focos diferentes:

    | | Claude | ChatGPT | Gemini |
    |---|---|---|---|
    | **Criador** | Anthropic | OpenAI | Google |
    | **Destaque** | Precisão, instruções longas | Versatilidade, plugins | Integração Google |
    | **Melhor para** | Código, análise detalhada | Uso geral | Gmail, Docs, Sheets |

    Para AI coding, recomendamos começar com **Claude** pela qualidade na geração
    de código e capacidade de seguir instruções complexas.

??? question "Preciso instalar algo no meu computador?"
    Para os primeiros módulos, **não**. Tudo funciona no navegador:

    - Claude funciona em claude.ai
    - Bolt funciona em bolt.new
    - Lovable funciona em lovable.dev

    Se você quiser usar o **Claude Code** (recomendado para projetos mais avançados),
    consulte o guia [Primeiros Passos](../primeiros-passos.md) para instalar VS Code,
    Git, Node.js e o Claude Code no seu computador.

??? question "Isso vai substituir programadores?"
    Não. Assim como o AutoCAD não substituiu engenheiros e arquitetos, AI coding
    não vai substituir desenvolvedores. O que muda é:

    - Programadores se tornam **mais produtivos** (menos trabalho repetitivo)
    - Não-programadores ganham a **capacidade de prototipar** e criar ferramentas simples
    - A barreira de entrada para criar software **diminui drasticamente**

    Projetos complexos, sistemas críticos e arquiteturas sofisticadas continuam
    precisando de profissionais especializados.
